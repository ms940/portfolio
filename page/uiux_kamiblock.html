KAMIBLOCK 3.0

*이 프로젝트는 대중적으로 알려진 블록코딩 ‘스크래치’ 프로그램을 기반으로 기업의 제품 기능에 맞춰 재구성한 UIUX 디자인을 진행했습니다.

작업 기여도

기획팀 개발 프로그래밍 100%
본인 기획 100%, 디자인 100%(개선 60% / 개발 40%)

주식회사 삼쩜일사는 로봇 ‘카미봇’을 제어하는 앱을 이용한 놀이학습으로 코딩의 기본기를 쉽게 공부하며 아이들이 텍스트 코딩까지 할 수 있게 하는 것을 최종 목표로 합니다.
프로젝트 Kamiblock 3.0은 텍스트 코드를 블록형으로 시각화시킨 것으로, 퍼즐처럼 끼워 맞추며 로봇을 제어하는 코딩 공부를 할 수 있게 합니다.

디자인

삼쩜일사가 개발한 하드웨어(코딩로봇 카미봇, 카미봇파이, x-block)를 연결해서 블록코딩을 학습할 수 있는 것에 집중했습니다. 기존 코딩 프로그램 스크래치의 개선이 필요한 점을 찾고 하드웨어 연동에 필요한 기능을 추가하는 작업을 진행했습니다.

OVERVIEW

1. 기존 앱의 개선사항을 찾아 해결 방안을 kamiblock 3.0에 반영
2. 하드웨어와 연결하여 사용 시 필요한 기능의 UI 추가

1
기존 앱 스크래치의 개선 사항

01 숨겨진 명령블록 버튼 표시

새로운 버튼을 추가하거나 기기환경에 따라 화면비율의 축소화에서 일부 버튼이 숨겨집니다. 이때 스크롤링 또는 스와이프 시 버튼이 나타나게 되지만 사전에 표시해 준다면 초기 사용자도 헤매지 않을 것입니다.
*현재는 스크래치 측에서 업데이트 되어 해결된 상황입니다.

개선 방안

그라데이션을 넣어 버튼이 추가적으로 더 있음을 사용자가 알 수 있도록 시각적으로 표시했습니다.

02 프로젝트명 표시와 작업영역 확보

진행중인 프로젝트가 어떤 파일인지 알기 위해 열린 파일명을 표시 해야합니다. 그리고 우측 패널로 인해 좁아진 작업 영역을 늘리기 위해서 패널 영역을 줄이는 방식을 선택했지만 여전히 작업 영역은 답답하며 작은 기기 사용시 더욱 불편합니다.

개선 방안

우측 패널의 레이아웃 정리로 작업 영역 공간을 확보하고 기본 상태의 공간을 조금 더 넓혀주었습니다. 기존 스크래치에서는 오른쪽 패널 사용성이 높았지만 이 프로젝트에서는 기업의 하드웨어와의 연동이 핵심입니다. 하드웨어를 연동한다면 왼쪽패널의 블록만 사용합니다. 때문에 축소 버튼을 숨김 버튼으로 대체했습니다. 또한 상대적으로 작아진 무대영역은 전체보기 기능으로 크게 볼 수 있기에 레이아웃 정리에 맞춰 과감히 줄였습니다.

03 입력창 배열 정리와 넘치는 텍스트에 대한 대응

스크래치의 우측 패널 입력 창 배열이 바르지 않은 이유는 언어 변환 시 텍스트 길이 변화에 따른 대응이였습니다.(패널비율은 고정) 그리고 요소 숨김/보이기 버튼 디자인이 입력창들 사이에서 이질감이 들었습니다. 언어 변환 시 넘치는 텍스트 문제를 해결하면서 통일감 있는 정리가 필요합니다.

개선 방안

텍스트 길이 폭 변동이 가장 큰 단어를 기준으로 여백의 기준점을 정하고 연관성 있는 입력창들은 간격을 두어 그룹돼 보이도록 했습니다. 요소 숨김/보이기 버튼은 작은 기기 사용 시 터치영역 확보를 위해 토글 버튼으로 변경하고 입력창들과 동일한 형태로서 통일감을 이루도록 했습니다.

04 테이블 여백 효율성

‘무대’란의 배경은 배경 사용 시 한 칸만 표시되고 좌측 상단의 ‘모양’ 버튼이 배경으로 바뀌면서 여러 배경을 관리하는 구조입니다. 배경은 한 칸만 표시되지만 세로 폭의 여백을 차지합니다. 아래 스프라이트 추가와 배경 추가 버튼은 각 테이블에 맞게 배치했지만 의도에 비해 명확하지 않으며 불필요한 여백을 감안하면서 배치를 좌우로 나누는 것은 비효율적입니다.

개선 방안

추가 기능 버튼은 클릭 시 세부 기능이 열립니다. 이미 기능별로 묶어서 나눠진 버튼이기에 해당 테이블에 맞춰 배치하지 않고 버튼끼리 묶어서 배치한다는 의도로 바꿨습니다. 또한 한 칸만 표시되는 배경 테이블을 아래로 가로 정렬하고, 오른쪽 남는 여백에 버튼들이 자연스럽게 배치되도록 했습니다. 스프라이트 테이블은 4열에서 3열로 줄여 가독성을 높였습니다.

2
하드웨어 추가 기능

확장기능 고르기 페이지

기업에서 개발한 3가지 모델의 하드웨어 제어를 통한 코딩이 가능합니다. 해당 페이지에서 선택한 하드웨어 연결을 시도하고 모델에 맞는 기능 블록들을 추가로 로드합니다.

하드웨어 관리창과 모니터링

연결된 하드웨어 목록과 연결 상태를 관리하는 기능과 하드웨어의 각 기능 센서들을 읽을 수 있는 모니터링 기능이 필요하여 추가했습니다. 실행된 창은 실시간으로 동작하며, 사용자가 집어서 이동하거나 줄이기가 가능합니다.
*내부 개발자들 의견을 모아 모달창으로 구현하기로 결정

하드웨어 제품군 중 모듈 형태의 X-block은 메인 두뇌 모듈에 센서별 모듈들을 연결해서 사용함으로, 모니터링 기능의 가장 많은 사용도를 차지합니다. 개발에 따라 추가되는 기능 모듈이 있으므로 그에 맞는 UI를 업데이트합니다.

Result

1. 이전 앱들을 통합하여 사용자와 개발자의 불편함 해소
2. 하드웨어 제품의 차별성과 경쟁력 부여

이미 회사에서 개발한 비슷한 앱들이 여럿 있었습니다. 사용자는 어떤 앱을 선택할지 곤란했고 개발자들은 개선 사항이 있을 땐 업무를 여러 번 하게 되는 불편함을 겪었습니다. kamiblock3.0은 모든 기능을 정리, 통합함으로써 사용자를 한 곳으로 모으고 개발자는 하나의 앱만 업데이트 하게 되었습니다. 또한 기업의 하드웨어를 연결한 다양한 코딩 교육을 진행할 수 있기에 경쟁력을 갖추고 스크래치의 기능 역시 구현 가능해 하드웨어가 없더라도 코딩 교육을 kamiblock3.0으로 모두 해결할 수 있게 됐습니다.
