KAMIBLOCK 3.0

*이 프로젝트는 대중적으로 알려진 블록코딩 ‘스크래치’ 프로그램을 기반으로 자사 카미봇의 기능에 맞춰 재구성한 UIUX 디자인을 진행했습니다.

작업 기여도

기획팀 개발 프로그래밍 100%
본인 기획 100%, 디자인 100%(개선 60%/개발 40%), 화면 설계 30%(기존 앱에서 리디자인)

주식회사 삼쩜일사는 어린이 코딩교육 로봇과 교육 커리큘럼을 개발합니다. 로봇 ‘카미봇’을 주 제품으로, 앱/프로그램을 사용하는 교육방법과 로봇의 기능만으로 교육할 수 있는 언플러그드 활동이 있습니다. 로봇 ‘카미봇’은 놀이학습으로 코딩의 기본기를 쉽게 공부하며 아이들이 텍스트 코딩까지 할 수 있게 하는 것을 최종 목표로 합니다. 프로젝트 Kamiblock 3.0 앱은 텍스트 코드를 블록형으로 시각화시킨 것으로, 퍼즐처럼 끼워 맞추며 카미봇의 기능을 제어하는 코딩 공부를 할 수 있게 합니다.

디자인

자사의 하드웨어 (코딩로봇 카미봇, 카미봇파이, 비트블럭 등)를 연동해서 블록코딩을 학습할 수 있는 것에 집중했습니다. 기존 코딩 프로그램 스크래치의 개선이 필요한 점을 찾고 하드웨어 연동에 필요한 기능을 추가하며 작업을 진행했습니다.

OVERVIEW

1. 기존 앱의 개선사항을 찾아 해결 방안을 kamiblock 3.0에 반영
2. 자사 개발 코딩로봇을 연동하여 사용 시 필요한 기능의 UI 추가

1
기존 앱 스크래치의 개선 사항

01 숨겨진 명령블록 버튼 표시

확장기능을 이용하면 추가 블록들을 불러오게 됩니다. 추가된 블록 버튼들이 늘어나면서 버튼 영역의 세로축 여백이 부족하게 되고 기존의 버튼들이 가려지게 됩니다. 추가 블록이 없더라도 기기에 환경에 따른 화면 비율이 줄어듦에 따라 버튼이 가려지는 경우가 생깁니다. 이때 스크롤링 또는 스와이프 시 버튼이 나타나게 되지만 버튼이 더 있음을 표시해 준다면, 초기 사용자도 헤매지 않을 것입니다.
*현재는 스크래치 측에서 업데이트 되어 해결된 상황입니다.

개선 방안

그라데이션을 넣어 아래 버튼이 추가적으로 더 있음을 사용자가 알 수 있도록 시각적으로 표시했습니다.

02 프로젝트명 표시와 작업영역 확보

어떤 것을 생산하는 프로그램 또는 앱을 사용한다면 이전 버전의 파일인지, 최종 파일을 수정하고 있는지 등 여러 파일을 사용하는 경우가 많으며 작업 시 확인해야만 할 사항이기에 파일명 또는 프로젝트명을 표시하는 부분이 추가되어야 합니다. 그리고 우측 패널 영역으로 인한 좁아진 작업 영역을 늘리기 위해 기존 앱에서는 패널을 줄이는 방식을 사용하지만 여전히 작업 영역은 답답하고 작은 기기 사용시 더욱 불편합니다.

개선 방안

우측 패널의 레이아웃 정리로 작업 영역 공간을 확보하여 축소 화가 아닌 상태일 때의 공간을 조금 더 넓혀주었습니다. 실제 앱을 사용할 때 작업 공간에 많은 블록들이 놓이게 되면 오른쪽 패널의 사용보다 작업 영역에서 더 오랜 작업이 이루어집니다. 특히 자사의 하드웨어를 연동한다면 오른쪽 패널은 사용성은 거의 무의미합니다. 때문에 오른쪽 패널을 축소하는 버튼을 패널 숨김 버튼으로 대체했습니다.

03 입력창 배열 정리와 넘치는 텍스트에 대한 대응

기존 프로그램의 우측 패널 입력 창의 배열이 바르지 않은 이유를 찾던 중, 여러 나라의 언어 변환 시 텍스트 길이의 변화에 따른 여백의 문제가 있었습니다. 그리고 요소 숨김/보이기 버튼 디자인이 입력창들 사이에서 이질감이 들었습니다. 언어 변환 시 넘치는 텍스트 문제를 해결하면서 통일감 있는 정리가 필요합니다.

개선 방안

텍스트 길이가 가장 긴 나라의 언어의 텍스트 길이 폭 변동이 가장 큰 단어를 기준으로 여백을 확보하고 서로 연관성 있는 입력창들은 간격을 두어 그룹으로 정리돼 보이도록 했습니다. 또한, 요소 숨김/보이기 버튼은 크기가 작아서 컴퓨터가 아닌 태블릿과 같은 작은 기기 사용 시 터치할 때 오류를 범할 수 있어 토글 버튼으로 변경하여 하나의 버튼으로 기능을 해결하며 디자인을 입력창들과 동일한 형태로 바꿔 통일감을 만들었습니다.

04 테이블 여백 효율성

‘무대’란의 배경은 배경 사용 시 한 칸만 표시되고 좌측 상단의 ‘모양’ 버튼이 배경으로 바뀌면서 여러 배경을 관리하는 구조입니다. 배경은 한 칸만 표시되지만 세로 폭의 여백이 불필요하게 차지하고 아래의 스프라이트 추가, 배경 추가 버튼들이 스프라이트 요소가 추가됨에 따라 겹쳐서 시각적 인식의 불편함이 있으며, 사용 기기가 작다면 버튼을 선택할 때 불편함이 생깁니다. 스프라이트와 무대의 각 테이블에 맞게 배치되어 있는 버튼이지만 의도에 비해 명확하지 않으며 위와 같은 문제점을 감안하면서까지 배치를 좌우로 나눠 비효율적입니다.

개선 방안

두 버튼은 눌렀을 때 세부 기능이 들어있습니다. 이미 기능별로 묶어서 나눠진 버튼이기에 테이블에 맞게 나눠서 배치하기보다는 하단으로 기능 버튼을 묶어서 배치한다는 의도로 바꿨습니다. 한 칸만 표시되는 배경 테이블을 아래로 내려 가로 배치하고 오른쪽 남는 여백에 버튼들을 자연스럽게 배치되도록 했습니다.

2
하드웨어 추가 기능

확장기능 고르기 페이지

자사에서 개발한 3가지 모델의 하드웨어 제어를 통한 코딩이 가능합니다. 해당 페이지에서 선택한 하드웨어 연결을 시도하고 모델에 맞는 기능 블록들을 추가로 로드합니다.

하드웨어 관리창과 모니터링

연결된 하드웨어 목록과 연결 상태를 관리하는 기능과 하드웨어의 각 기능 센서들을 읽을 수 있는 모니터링 기능이 필요하여 추가했습니다. 실행된 창은 실시간으로 동작하며, 사용자가 집어서 이동하거나 줄이기가 가능합니다.

하드웨어 제품군 중 모듈 형태의 X-block은 메인 두뇌 모듈에 센서별 모듈을 여러 개수를 사용하는 것으로, 모니터링 기능의 가장 많은 사용도를 차지합니다. 개발에 따라 추가되는 기능 모듈이 있으므로 그에 맞는 UI를 업데이트합니다.

Result

1. 이전 앱들을 통합하여 사용자와 개발자의 불편함 해소
2. 자사 제품의 차별성과 경쟁력 부여

자사에서 개발한 비슷한 앱들이 여럿 있었습니다. 사용자는 어떤 앱을 선택할지 곤란했고 개발자들은 개선 사항이 있을 땐 업무를 여러 번 하게 되는 불편함을 겪었습니다. kamiblock3.0 앱에 모든 기능을 정리해서 통합함으로써 이와 같은 문제점들을 해결하고, 자사의 제품을 연동하여 더 다양한 코딩 교육을 진행할 수 있기에 자사 제품의 차별성, 경쟁력을 높였습니다. 스크래치의 기능 또한 구현할 수 있어 자사의 제품이 없더라도 코딩 교육이 가능합니다. 회원 한정이지만 자사 공식 카페에 kamiblock3.0을 이용한 콘텐츠를 선생님들과 부모님에게 제공함으로써 그 사용성은 한몫 더 빛을 발했습니다.
